Linux下musl-libc的安装

发表于 2023-11-16 更新于 2023-11-17 分类于 Tech

前言

Linux下默认的c标准库是glibc，glibc是GNU项目的一部分，由于在很长的历史发展中，glibc为了性能优化、兼容性问题，加入了很多复杂的汇编优化及冗余代码，因此可读性不高，在一些程度上不利于新手对libc的学习。与此同时，目前有很多开源的libc代码，由于没有历史包袱，因此可读性非常高。下面以printf的实现为例，展示musl libc的实现。

musl libc简介

musl是一个C标准库，主要用于基于Linux内核的操作系统，使用MIT许可证发布。它由RichFelker开发，目标是编写一个干净、高效、符合标准的libc实现。

官方网址：http://musl.libc.org

参考文档：musl libc Reference Manual

musl更多介绍：About musl

需要注意，目前musl libc需要在Linux内核的系统上配置。

运行配置文件的格式是

[OPTION]和[VAR=VALUE]可用于指定configure配置的选项和参数，以下是常用的例子：

  • –prefix，用于指定musl的安装目录，默认是/usr/local/musl，而不是/usr/local，这是为了避免与主机上现有的非musl环境冲突。如果想要安装musl作为系统libc，则前缀设置为/usr或/。

  • –syslibdir，用于指定动态链接器的安装位置。默认是/lib。

  • CC=…，指定编译器，默认将是系统上默认的编译器，如gcc。

  • CFLAGS=…，编译器的自定义选项。

  • –disable-shared，默认不开启，开启则禁用构建共享的libc.so，将减半构建时间。

  • –disable-static，默认不开启，开启则禁用构建libc.a，但其他.a文件仍然被构建。这也将减半构建时间。

  • –enable-warnings，打开推荐的GCC警告选项集，musl旨在编译无警告。

  • –enable-debug，打开调试。在CFLAGS中手动添加-g也可以。在未来，–enable-debug可能还会在源代码级别启用额外的调试功能。这一项调试程序时有显著的作用。例如，当使用gdb调试一个c程序时，如果在编译过程加入-g选项，gdb调试仅仅只能在当前编译的c文件中调试，而不能调试进入c的库函数程序。如果加入这一选项，调试过程可以深入库函数内部（例如printf、malloc等），了解程序每一步的运行的状态。

运行../configure --help可以看到configure配置的更多参数。

关于交叉编译

交叉编译是一种编译技术，它允许开发者在一个平台（称为主机）上生成另一个平台（称为目标）上运行的代码。例如，本文的背景是在macOS上编译一个Linux可执行文件。

在这个背景下，主机是x86_64架构的macOS，目标是x86_64架构的Linux。因此需要一个交叉编译器，这个编译器在macOS上运行，但编译生成的代码是为Linux准备的，不能在主机上运行。

以下是一个简单的步骤，说明如何在macOS上交叉编译Linux可执行文件：

  1. 构建交叉编译器：需要通过一些途径构建主机上的交叉编译工具，例如x86_64-linux-gnu-gcc。

  2. 编写代码：像平时一样编写你的C或C++代码，代码应该不依赖于任何特定于macOS的特性或库，因为这些在Linux上可能不存在。

  3. 编译代码：使用交叉编译器来编译你的代码。例如，你可以使用以下命令来编译一个名为hello.c的C程序：

    这个命令会生成一个名为hello的可执行文件。

  4. 测试代码：你可以把生成的可执行文件复制到一个Linux机器上，也可以是虚拟机或Docker，然后运行它来测试。

这只是一个基本的介绍。交叉编译可能会涉及到更复杂的问题，例如处理库依赖、调试和优化等。以下主要介绍的是这里的第一步，也就是如何构建交叉编译工具。

关键词过滤喵，谢谢喵（flag1，flag2）

1、flag1需要统计字符数量，按照以下思路即可完成

  • 把所有字符替换成统一字符，例如“A”

  • 如果没有“A”就替换一个0，输出即可结束

  • 把第一个“A”替换成“0A”，“0”代表现在开始统计了0个字符

  • 把0-8数字+A的组合替换成大1的数字，这里需要写八条指令，意思是把个位数加1，并去掉一个字符

  • 如果数字是9，需要进位，进入进位模块

  • 进位需要引入一个进位符，表示进1位，例如B，进位时，需要不断把“9A”替换成“0B”，这可以用于遇到多次进位时不断进位。

  • 最终如果没有“A”了，就可以结束

2、因为在这个问题中只有换行符是可以自由替换，不影响结果的，所以需要灵活运用换行符的替换。思路如下

  • 把所有换行符去除，并在开头、末尾和每两行之间插入两个空行

  • 开始时，把每一行的第一个字符后加入换行，这样所有文本之间都空了两行，每一段文本分割成连续的两行，第一行表示已经统计的文本，第二行表示尚未统计的，此后每一次都将第二行的第一个字符放到第一行末尾

  • 如果遇到出现连续3个空行，说明已经有一行文本统计完毕，这是最短的那一行文本，将其放在最后一行（这一步需要多次判断，以防有同样长度的），与还在统计的文本用4个换行隔开，以防之前的操作混乱

  • 之后继续重复，每遇到一次三连换行就将这一换行前的文本放置最后

  • 最后如果没有连续的两行，说明统计结束，将所有换行去掉即可

(#prob08-cookie)

本题原理是基于python随机数的算法是MT19937，这一算法根据前624个8字节的数可以确定状态，从而预测后续的随机数。本题直接采用以开发好的python第三方库randcrack。

1、刚开始搞错了位数，代码经过了一些无脑修改，写的质量不高。思路如下：

  • words的前2500个字符是0，因此与随机数异或运算后保留的是随机数本身，通过这些字节预测后续随机数，在异或回去就可以得到words

2、第二题类似，主要利用的是异或的交换律、结合律、逆运算为本身。在这里虽然设置了熵对随机数进行了偏移，但是测试可以发现随机数异或后不会影响结果。

  • 用随机数种子产生两个随机数状态，熵从0开始计算，每一次按照words和输出结果的关系进行异或运算。

3、利用zip参数长度不一致时只会取最短的那个，复制输入第一个十六进制的随机数就能通过。

华维码（flag1）

(# prob19-qrcode)

本题无需在网页上通过华容道，直接拼出二维码扫描即可。本题使用python第三方库pyzbar进行二维码扫描，思路分两步。

  • 通过浏览器抓包功能下载24张图片，可以发现缺少的是二维码左上的图，直接旋转一张右上或左下的图即可

  • flag1是version 2的二维码。按照二维码的规范，可以确定16张图片的位置。剩下的9张图片，第4行第5个图片只有两种可能性。右下角的那一张必须左上方是黑点。排列组合后一共有65520种情况。

  • 使用python的pillow进行图片拼接，并逐一扫描即可获得答案。
